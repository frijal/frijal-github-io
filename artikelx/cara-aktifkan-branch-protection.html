Cara Mengaktifkan Branch Protection di GitHub

oleh Fakhrul Rijal

Panduan ini akan menunjukkan cara mengaktifkan branch protection di GitHub agar semua perubahan wajib melalui Pull Request (PR) dan tidak ada yang bisa melakukan push langsung ke branch `main`.

1. Masuk ke Pengaturan Repository

Buka repository Anda di GitHub, lalu klik tab Settings.

2. Pilih Menu Branches

Di menu navigasi sebelah kiri, pilih Branches. Kemudian, pada bagian "Branch protection rules", klik tombol Add rule.

3. Tentukan Branch yang Ingin Dilindungi

Pada kolom Branch name pattern, ketik nama branch utama Anda, yaitu main. Anda juga bisa menggunakan wildcard jika ingin melindungi beberapa branch sekaligus, misalnya release/*.

4. Aktifkan Aturan Proteksi Kunci

Centang beberapa opsi penting berikut untuk memperkuat keamanan branch Anda:

  • Require a pull request before merging: Ini adalah aturan paling fundamental. Semua perubahan wajib diajukan melalui Pull Request.
  • Require status checks to pass before merging: Jika Anda menggunakan workflow CI/CD (seperti GitHub Actions), aturan ini memastikan PR hanya bisa di-merge jika semua tes berhasil.
  • Include administrators: Aturan ini sangat direkomendasikan. Dengan mencentangnya, bahkan admin atau pemilik repository pun harus mengikuti aturan yang sama (tidak bisa push langsung).

5. Simpan Aturan

Setelah selesai, gulir ke bawah dan klik tombol Create atau Save changes untuk menyimpan aturan proteksi Anda.

Hasil Akhir

Setelah aturan aktif, Anda akan melihat hasilnya secara langsung:

  • Setiap upaya untuk melakukan push langsung ke branch main akan diblokir oleh GitHub.
  • Semua perubahan, sekecil apa pun, harus diajukan melalui Pull Request dan akan tunduk pada aturan yang telah Anda tetapkan (misalnya, memerlukan review atau status check).
