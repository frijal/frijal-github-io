Fenomena pengguna biasa yang mulai "menelan pil pahit" karena kurangnya aplikasi tertentu (seperti aplikasi UWP/APPX dari Windows Store) dan akhirnya melirik Linux semakin nyata. Mereka mencari kebebasan, keamanan, dan efisiensi yang Linux tawarkan.

Namun, muncul jurang pemisah: pengguna awam ini sama sekali tidak ingin, dan sebenarnya tidak harus, menyentuh Terminal. Mereka terbiasa dengan pengalaman point-and-click ala Windows atau macOS. Mereka hanya ingin semuanya berjalan mulus dengan klik dan GUI.

Jurang Pemisah: Harapan GUI vs. Realitas CLI

Bagi pengguna awam, Terminal adalah momok yang menakutkan. Padahal, di banyak distro Linux, Terminal masih jadi andalan untuk hal penting:

  • Mengatasi bug kecil (audio, driver, error paket).
  • Instalasi software non-standar.
  • Manajemen sistem & upgrade besar.

Misi Desainer Distro: Full Mouse Experience

Distro masa depan harus menawarkan pengalaman 100% mouse-driven. Artinya, dari instalasi sampai penggunaan, pengguna tidak dipaksa buka Terminal.

1. Zero-CLI Installation & Setup

Pusat aplikasi harus bisa pasang DEB, RPM, Flatpak, Snap dengan klik. Driver otomatis terdeteksi tanpa repot.

2. Mengisi Kekosongan Aplikasi Windows

Integrasi Wine/Proton harus semudah klik install, bukan konfigurasi panjang di Terminal.

3. Menutup Gap GUI

Masalah Solusi GUI
Drive eksternal NTFS error Tombol "Perbaiki Drive" di File Manager
Resolusi layar eksternal rumit GUI multi-monitor drag-and-drop
Update kernel rusak driver Tombol rollback kernel di Settings

Dengan pendekatan ini, Linux bisa benar-benar jadi ramah untuk pengguna biasa.
