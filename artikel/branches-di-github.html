Branch (cabang) adalah jalur pengembangan terpisah di dalam sebuah repositori Git. Ibaratnya, kamu punya pohon kode: batang utamanya adalah default branch (biasanya bernama main), lalu kamu bisa membuat cabang baru untuk mengembangkan fitur, memperbaiki bug, atau bereksperimen tanpa mengganggu batang utama.

Fungsi dan Maksud Branch

  • Isolasi pekerjaan: kamu bisa mengerjakan fitur baru tanpa merusak kode utama yang stabil.
  • Kolaborasi lebih aman: tiap developer bisa bekerja di branch masing-masing, lalu menggabungkannya (merge) lewat pull request.
  • Eksperimen bebas risiko: kalau gagal, cukup hapus branch tanpa merusak kode utama.
  • Manajemen versi: memungkinkan adanya branch khusus untuk berbagai keperluan, seperti develop, feature/*, atau bugfix/*.

Alur Umum Penggunaan

  1. Buat branch baru dari main:
    2. git checkout -b fitur-login
  2. Kerjakan perubahan di branch tersebut (tambah/edit file).
  3. Push ke GitHub:
    4. git push origin fitur-login
  4. Ajukan pull request di GitHub untuk menggabungkan ke main.

Bayangkan kamu menulis buku. Naskah utama yang sudah rapi adalah main. Saat ingin menambah bab baru, kamu membuat branch sebagai draf alternatif. Kalau draf itu bagus, kamu gabungkan ke naskah utama. Kalau tidak, buang saja drafnya tanpa merusak buku utama.

Praktik Terbaik âœ¨

  • Beri nama branch yang jelas: feature/nama-fitur, bugfix/123-deskripsi-singkat.
  • Aktifkan proteksi branch main: wajibkan adanya review sebelum merge.
  • Hapus branch setelah selesai: jaga repositori tetap bersih setelah pull request di-merge.
