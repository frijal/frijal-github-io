Bagi pengguna Fedora dengan kartu nirkabel **Broadcom** (seri bcm43xx), seringkali terjadi masalah koneksi Wi-Fi terputus setelah melakukan **update kernel** terbaru. Ini biasanya karena modul kernel *bcma* bawaan mengganggu driver Broadcom yang sudah terpasang. Panduan singkat ini akan menunjukkan cara cepat dan mudah untuk mengatasi masalah tersebut.

Langkah 1: Nonaktifkan Modul Kernel Konflik (bcma)

Langkah pertama adalah membuat daftar hitam (*blacklist*) untuk modul kernel `bcma`. Ini mencegah kernel memuatnya saat *booting*, sehingga memberi ruang bagi driver Broadcom yang benar.

1.1. Buka Terminal dan Akses Root

Buka terminal dan masuk sebagai **root** atau gunakan `sudo -i` untuk mendapatkan hak akses penuh:

sudo -i

1.2. Buat File Blacklist

Jalankan perintah berikut untuk membuat file konfigurasi baru yang berisi perintah *blacklist* untuk modul `bcma`.

echo "blacklist bcma" >> /etc/modprobe.d/broadcom-wl-blacklist.conf

üí° **Tips:** Perintah `>>` akan menambahkan baris baru ke akhir file tanpa menghapus konten yang sudah ada. Jika file tersebut belum ada, perintah ini akan membuatnya.

Langkah 2: Terapkan Perubahan dan Restart

Setelah modul kernel yang bermasalah telah di-*blacklist*, Anda perlu meregenerasi initramfs (gambar RAM awal) dan me-reboot komputer agar perubahan diterapkan sepenuhnya.

2.1. Regenerasi Initramfs (Opsional tapi Disarankan)

Meskipun langkah *reboot* biasanya cukup, menjalankan perintah ini memastikan sistem sepenuhnya menyadari perubahan *blacklist*:

dracut -f

2.2. Reboot Sistem

Restart komputer Anda. Setelah sistem kembali menyala, koneksi nirkabel Broadcom seharusnya sudah berfungsi normal dan stabil, bahkan setelah update kernel di masa depan.

reboot
