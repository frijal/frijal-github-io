Mengatasi LibreOffice yang Lambat di Jaringan

Pernah mengalami saat membuka dokumen LibreOffice dari folder jaringan, prosesnya sangat lambat? Masalah ini cukup umum, terutama bagi pengguna Linux. Ternyata, solusinya cukup sederhana.

Akar Masalah: GIO vs GVFS-Fuse

Kelambatan ini sering terjadi karena fitur GIO (Glib Input/Output) yang terlalu "cerewet" saat mengakses file di jaringan. Solusinya adalah menonaktifkan fitur tersebut dan memaksa LibreOffice untuk langsung menggunakan GVFS-Fuse, yang umumnya lebih cepat untuk kasus ini.

Langkah-langkah Perbaikan

1. Pastikan gvfs-fuse Sudah Terpasang

Buka terminal dan jalankan perintah berikut untuk memastikan paket yang diperlukan sudah ada di sistem Anda.

sudo apt-get install gvfs-fuse

2. Nonaktifkan GIO untuk LibreOffice

Selanjutnya, jalankan perintah sed ini di terminal untuk mengedit file konfigurasi LibreOffice secara otomatis. Perintah ini akan memberi komentar pada baris yang mengaktifkan GIO.

sudo sed -i 's/X-GIO-NoFuse=true/#X-GIO-NoFuse=true/' /usr/share/applications/libreoffice*

Setelah menjalankan kedua perintah tersebut, coba buka kembali dokumen Anda dari jaringan. Seharusnya, LibreOffice akan merespons jauh lebih cepat.

Inti Masalah

  • LibreOffice lambat saat membuka file dari folder jaringan (SMB/NFS).
  • Penyebabnya adalah fitur GIO yang kurang efisien untuk beberapa koneksi.

Inti Solusi

  • Langkah 1: Instal gvfs-fuse.
  • Langkah 2: Nonaktifkan GIO dengan perintah sed.
  • Hasil: LibreOffice akan menggunakan GVFS-Fuse yang lebih cepat.
