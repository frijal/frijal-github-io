• Link Membuka di Tab yang Sama: Karena tidak ada target="_blank", link akan terbuka di tab atau jendela yang sedang aktif, menggantikan halaman saat ini.
  • Perilaku Link Normal: Selain menyembunyikan referrer, link akan berfungsi seperti link biasa.
  • Referrer Disembunyikan: Saat pengguna mengklik link, situs tujuan tidak akan mengetahui dari halaman mana pengguna tersebut berasal. Informasi Referer pada HTTP header akan dihilangkan.
  • Tidak Ada Proteksi window.opener: Atribut rel="noopener" (yang sering dipasangkan dengan noreferrer) berfungsi untuk mencegah halaman baru mendapatkan akses ke objek window dari halaman asal. Namun, karena link ini tidak membuka tab baru, ancaman keamanan terkait window.opener tidak relevan.

    • Dampak rel="noreferrer" Tanpa target="_blank"

    Dampak yang Tetap Ada (Privasi)

    Dampak yang Tidak Terjadi (Keamanan)

    Kesimpulan

    Panduan santai ini akan menjelaskan tentang dampak privasi dan keamanan saat menggunakan atribut rel="noreferrer" pada link HTML tanpa target="_blank".

    Jadi, rel="noreferrer" tetap berguna meskipun tanpa target="_blank". Bedanya, dalam kasus ini, efeknya murni berfokus pada privasi (menyembunyikan sumber traffic), bukan pada keamanan yang terkait dengan pembukaan tab baru.

    Jika Anda menggunakan rel="noreferrer" tetapi tidak menyertakan target="_blank", efek privasi tetap berlaku. Artinya:
