‚ö° Panduan Memilih Repository Linux Tercepat

Repository (atau mirror) adalah server tempat sistem operasi Linux mengunduh paket aplikasi. Memilih repository yang tercepat dan terdekat dari lokasi kita akan membuat proses update dan instalasi aplikasi jadi lebih cepat.

üîë Kunci utama: Gunakan repository resmi dari distro yang kamu pakai, lalu pilih mirror yang paling dekat dengan lokasi geografismu.

1. Keluarga Debian (Debian, Ubuntu, Linux Mint)

Distro ini menggunakan format paket .deb dan dikelola oleh APT. Cara memilih mirror tercepat:

  • Debian: edit file /etc/apt/sources.list ‚Üí pilih mirror dari daftar mirror Debian.
  • Ubuntu: gunakan Software & Updates GUI ‚Üí pilih "Download from ‚Üí Other ‚Üí Select Best Server".
  • Linux Mint: di "Software Sources" ‚Üí pilih mirror tercepat untuk "Main" dan "Base".
    
# Contoh mengganti ke mirror Indonesia (Debian)
deb http://kartolo.sby.datautama.net.id/debian/ bookworm main contrib non-free

2. Keluarga Arch (Arch Linux, Manjaro)

Arch dan turunannya memakai paket .pkg.tar.zst dengan pacman. Pilih mirror dengan tools bawaan:

  • Arch Linux: gunakan reflector untuk memilih mirror tercepat.
  • Manjaro: gunakan pacman-mirrors untuk generate daftar mirror baru.
    
# Arch Linux: pilih 5 mirror tercepat dari negara tertentu
sudo reflector --country "Indonesia" --latest 5 --sort rate --save /etc/pacman.d/mirrorlist

# Manjaro: refresh mirrorlist
sudo pacman-mirrors --fasttrack && sudo pacman -Syyu

3. Fedora (RPM-based)

Fedora menggunakan format paket .rpm dengan manajer dnf. Repository default biasanya sudah otomatis memilih mirror terbaik. Jika ingin manual, edit file di /etc/yum.repos.d/fedora.repo. 

    
# Tes kecepatan mirror Fedora
sudo dnf makecache --refresh

4. openSUSE (RPM-based)

openSUSE juga menggunakan .rpm, dengan manajer zypper. Untuk memilih mirror, gunakan layanan resmi openSUSE MirrorBrain yang otomatis mengarahkan ke server terdekat. 

    
# Refresh semua repository openSUSE
sudo zypper refresh

5. NixOS

NixOS menggunakan sistem unik berbasis nix. Mirror dapat dipilih dengan mengedit konfigurasi di /etc/nix/nix.conf. Daftar mirror tersedia di situs resmi NixOS.

6. Puppy Linux

Puppy Linux biasanya berbasis Ubuntu atau Slackware, sehingga mengikuti mirror induknya. Jika berbasis Ubuntu ‚Üí gunakan mirror Ubuntu. Jika berbasis Slackware ‚Üí gunakan mirror Slackware.

7. Slackware

Slackware menggunakan paket .tgz atau .txz. Gunakan slackpkg untuk mengelola repository, lalu pilih mirror di file /etc/slackpkg/mirrors. 

    
# Edit daftar mirror Slackware
sudo nano /etc/slackpkg/mirrors

Kesimpulan

  • Debian/Ubuntu/Mint ‚Üí APT (.deb) ‚Üí pilih mirror terdekat via GUI/daftar resmi.
  • Arch/Manjaro ‚Üí Pacman (.pkg.tar.zst) ‚Üí gunakan reflector / pacman-mirrors.
  • Fedora/openSUSE ‚Üí DNF/Zypper (.rpm) ‚Üí mirror otomatis + bisa edit repo file.
  • NixOS ‚Üí Nix (.nar) ‚Üí atur di nix.conf.
  • Puppy Linux ‚Üí mengikuti basis (Ubuntu/Slackware).
  • Slackware ‚Üí Slackpkg (.tgz/.txz) ‚Üí edit /etc/slackpkg/mirrors.

Dengan memilih repository tercepat sesuai distro dan lokasi, proses update, upgrade, dan instalasi aplikasi akan jauh lebih cepat ‚ö°.
