  • Branch protection: Ini fitur opsional yang tujuannya untuk menambah aturan internal, terutama kalau kamu bekerja dalam tim dengan banyak kolaborator.
  • Buat Pull Request (PR), yaitu mengusulkan perubahan. Tapi, kamu tetap punya kendali penuh untuk memutuskan apakah usulan itu akan diterima (di-merge) atau tidak.
  • Fork repo kamu, alias bikin salinan repo-nya di akun mereka sendiri.
  • GitHub Actions: Bot atau skrip otomatis hanya bisa push kalau kamu memberikan izin (`contents: write`) menggunakan `GITHUB_TOKEN` atau Personal Access Token (PAT) yang kamu buat sendiri.
  • Kamu (pemilik repo): Jelas, kamu selalu punya hak penuh untuk push.
  • Kamu (atau bot yang kamu izinkan) bisa push langsung ke branch main tanpa hambatan.
  • Kolaborator: Orang-orang yang sudah kamu undang secara khusus sebagai kolaborator juga bisa push ke repo.
  • Repo privat: Hanya kamu dan kolaborator yang kamu undang yang bisa mengakses.
  • Repo publik: Orang lain bisa melihat dan meng-fork, tapi tidak bisa push langsung.
  • Tidak ada aturan tambahan yang wajib dipenuhi, misalnya harus ada review dari kolaborator lain atau harus lulus tes otomatis dulu sebelum kode bisa di-merge.
