Apa Artinya 'Main Branch Isn't Protected' di GitHub? ü§î

Sering banget kan, pas buka repository di GitHub, tiba-tiba muncul peringatan kayak gini: "Your main branch isn't protected, Protect this branch from force pushing or deletion, or require status checks before merging." Nah, kalau lihat tulisan itu, banyak yang langsung mikir, "Waduh, berarti semua orang bisa seenaknya ubah dan hapus file di repo saya dong?"

Tenang, itu pertanyaan yang bagus dan penting banget buat dipahami. Jawabannya: tidak.

Meskipun branch utama kamu tidak dilindungi (unprotected), bukan berarti semua orang bisa seenaknya ubah atau hapus file di repo kamu. Mari kita bahas lebih lanjut biar kamu makin paham.

---

üîë Siapa Saja yang Bisa Push ke Repo Kamu?

Ada aturan ketat di GitHub tentang siapa yang bisa langsung mengubah kode di repo kamu. Ini dia daftarnya:

  • Kamu (pemilik repo): Jelas, kamu selalu punya hak penuh untuk push.
  • Kolaborator: Orang-orang yang sudah kamu undang secara khusus sebagai kolaborator juga bisa push ke repo.
  • GitHub Actions: Bot atau skrip otomatis hanya bisa push kalau kamu memberikan izin (`contents: write`) menggunakan `GITHUB_TOKEN` atau Personal Access Token (PAT) yang kamu buat sendiri.

Poin paling pentingnya adalah: Orang luar (publik, user lain di GitHub) tidak bisa langsung push ke repo kamu. Mereka cuma bisa melakukan dua hal ini:

  • Fork repo kamu, alias bikin salinan repo-nya di akun mereka sendiri.
  • Buat Pull Request (PR), yaitu mengusulkan perubahan. Tapi, kamu tetap punya kendali penuh untuk memutuskan apakah usulan itu akan diterima (di-merge) atau tidak.

üîí Apa Maksud 'Main Branch Isn‚Äôt Protected'?

Jadi, arti sebenarnya dari peringatan itu lebih ke soal aturan internal. Peringatan itu cuma memberitahu kamu bahwa:

  • Kamu (atau bot yang kamu izinkan) bisa push langsung ke branch main tanpa hambatan.
  • Tidak ada aturan tambahan yang wajib dipenuhi, misalnya harus ada review dari kolaborator lain atau harus lulus tes otomatis dulu sebelum kode bisa di-merge.

Jadi, peringatan itu sama sekali tidak berarti publik bisa bebas push atau hapus kode. Ini lebih seperti "jalan bebas hambatan" untuk pemilik repo sendiri.

---

üìå Ringkasnya

  • Repo publik: Orang lain bisa melihat dan meng-fork, tapi tidak bisa push langsung.
  • Repo privat: Hanya kamu dan kolaborator yang kamu undang yang bisa mengakses.
  • Branch protection: Ini fitur opsional yang tujuannya untuk menambah aturan internal, terutama kalau kamu bekerja dalam tim dengan banyak kolaborator.

Intinya, kalau repo ini hanya kamu yang pegang, membiarkan branch `main` tidak terlindungi itu tidak masalah sama sekali. Aman kok! üòâ
