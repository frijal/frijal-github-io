Refresh GitHub Codespaces dari Browser

Rebuild, full rebuild, restart, atau stop-start—kapan pakai yang mana. Disajikan dalam format satu kolom yang nyaman untuk semua perangkat.

Diperbarui: 06 Okt 2025

Panduan Rebuild Codespaces

Penjelasan singkat dan langsung bisa dipakai. Berikut adalah langkah-langkah praktis untuk me-refresh environment Codespaces Anda.

🔄 Cara Rebuild Codespace dari Browser

  1. Buka repositori Anda di GitHub, lalu klik tab **Codespaces** → pilih codespace yang mau dibuka.
  2. Tekan Ctrl+Shift+P (Windows/Linux) atau Cmd+Shift+P (Mac) untuk membuka *Command Palette*.
  3. Ketik `Rebuild` → pilih salah satu:
    • Codespaces: Rebuild Container → rebuild cepat dengan cache.
    • Codespaces: Full Rebuild Container → rebuild total dari nol.
  4. Tunggu proses selesai. File kode Anda tetap aman, hanya environment yang di-refresh.

✨ Perbedaan Rebuild vs Full Rebuild

Mode Apa yang dilakukan Kapan dipakai
Rebuild Menggunakan cache image lama untuk mempercepat proses. Perubahan kecil di devcontainer.json atau saat butuh refresh cepat.
Full Rebuild Hapus semua cache dan bangun ulang container dari awal. Saat environment error, dependency rusak, atau ingin memastikan setup benar-benar fresh.

⚡ Tips Tambahan

Aksi Keterangan
Restart Codespace Mirip restart komputer, lebih cepat dari rebuild. Akses dari menu kiri bawah.
Stop & Start Mematikan dan menyalakan ulang dari dashboard GitHub. Berguna jika sesi "stuck".
