Mengatasi Partisi NTFS Tidak Bisa Dibuka di Caja File Manager

Pada beberapa distro Linux dengan Caja File Manager, partisi NTFS terkadang gagal dibuka. Biasanya masalah ini terkait dengan pengaturan udisks2 yang masih menggunakan aturan auth_admin sehingga akses hanya diberikan jika ada otorisasi admin.

Pesan Error di Caja

Hal ini biasanya disebabkan oleh file policy org.freedesktop.udisks2.policy yang masih default (auth_admin).

Solusi

Buka terminal, lalu edit file org.freedesktop.udisks2.policy menggunakan perintah:

sudo nano /usr/share/polkit-1/actions/org.freedesktop.udisks2.policy

Cari bagian berikut:

<action id="org.freedesktop.udisks2.filesystem-mount-system">
  <description>Mount a system filesystem</description>
  <message>Authentication is required to mount the filesystem</message>
  <defaults>
    <allow_any>auth_admin</allow_any>
  </defaults>
</action>

Ubah auth_admin menjadi yes:

<allow_any>yes</allow_any>

Simpan perubahan, lalu coba buka kembali partisi NTFS. Sekarang partisi seharusnya sudah bisa di-mount tanpa kendala.
