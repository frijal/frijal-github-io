Narasi asli, santai, dan lengkap

Ingin aplikasi favoritmu otomatis berjalan saat login ke Ubuntu? Dulu kamu bisa menggunakan alat Startup Applications, tapi di Ubuntu 25.10, alat itu sudah tidak ada lagi.

Ubuntu 25.10 hadir dengan GNOME 49 dan versi baru paket gnome-session. Paket inilah yang sebelumnya menyediakan GUI Startup Applications, memudahkan pengguna menambahkan perintah khusus, menjalankan skrip, atau daemon setiap kali login.

Namun, GNOME 49 membawa banyak perubahan dalam cara sesi ditangani, termasuk integrasi lebih dalam dengan systemd, sehingga frontend gnome-startup-applications dihapus dari Ubuntu 25.10.

Meski begitu, hilangnya frontend tidak berarti kemampuan autostart aplikasi ikut hilang. Pengguna tetap bisa mengatur aplikasi yang berjalan otomatis lewat Settings â†’ Applications. Pilih aplikasi yang ingin dijalankan (atau dihentikan) saat login, lalu aktifkan/nonaktifkan toggle Autostart.

Penulis menambahkan bahwa mungkin tidak banyak pengguna Ubuntu yang benar-benar memakai Startup Applications (karena GNOME sendiri sudah lama mengarahkan orang ke GNOME Tweaks). Jadi kemungkinan besar hanya sedikit yang menyadari alat ini hilang.

Yang dulu bisa:

  • Menambahkan perintah khusus
  • Menunjuk ke skrip
  • Menjalankan daemon
  • Menambahkan flag/argumen ke peluncur GUI

Sekarang: cara GUI untuk hal-hal itu tidak lagi tersedia di Ubuntu 25.10. Jika kamu mencari Startup Applications dan bertanya kenapa tidak bisa diinstall lagi dengan sudo apt install gnome-startup-applicationsâ€”ini alasannya.

Contoh praktis: file .desktop untuk autostart

Untuk fleksibilitas penuh, kamu bisa membuat file .desktop di ~/.config/autostart/. Contoh di bawah menjalankan sebuah skrip saat login:

[Desktop Entry]
Type=Application
Version=1.0
Name=Jalankan Skrip Login
Comment=Menjalankan skrip kustom saat masuk
Exec=/home/<user>/scripts/startup.sh --verbose
StartupNotify=false
Terminal=false
X-GNOME-Autostart-enabled=true

Tip: pastikan skrip kamu memiliki izin eksekusi (chmod +x) dan gunakan path absolut. Jika butuh penundaan, bungkus dengan bash -lc atau tambahkan sleep di dalam skrip.
