Hapus Kernel Lama di Linux

Dark-mode 1 kolom, fullscreen, responsif, interaktif, dan SEO lengkap.

Kernel lama numpuk? Bisa dibersihkan. Tapi lakukan dengan aman: cek dulu versi yang terpasang, dan biasakan menyimpan minimal dua kernel terakhir untuk jaga-jaga kalau versi baru bermasalah. Di bawah ini ada perintah praktis dan alasan kenapa fallback itu penting.

Inti pembahasan dari thread

  • Tanya: How to remove old kernels? How to remove completely and keep only the latest one?
  • Jawab santai: Tampilkan semua kernel dan headers terpasang dengan:
    dpkg -l "linux-image*" "linux-headers*"
  • Hapus spesifik: Kamu bisa menghapus paket kernel tertentu (beserta konfigurasi) dengan:
    sudo apt purge linux-image-6.12.2-amd64
  • Lebih aman: Biasanya lebih baik pakai:
    sudo apt autoremove
    untuk menghapus otomatis semua kecuali dua versi terakhir.
  • Autopurge: Jika kernel di-install via dependency linux-image-amd64, jalankan:
    sudo apt autopurge
    ini akan menyisakan dua kernel terbaru sekaligus membersihkan sisa konfigurasi.

Kenapa menyimpan dua kernel terakhir itu bijak?

  • Bug terjadi: Kadang kernel baru buggy‚Äîfallback penting agar kamu bisa boot ke versi terakhir yang stabil.
  • Hardware bleeding edge: GPU atau perangkat sangat baru sering butuh waktu sampai kode stabil hadir; ekstra kernel membantu.
  • Kebersihan sistem: Jalankan apt autoclean sesekali untuk bersihkan cache paket, tanpa mengganggu kernel aktif.
  • Status dpkg: Baris ‚Äúii‚Äù artinya paket terpasang, ‚Äúrc‚Äù artinya sudah di-remove tapi belum di-purge (masih ada sisa config). apt purge akan menuntaskan sisa tersebut.
Best practice: simpan minimal dua kernel terakhir. Jangan purge kernel yang sedang aktif. Selalu cek daftar versi sebelum menghapus.
